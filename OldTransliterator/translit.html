There are two panes above. Left pane is for Romanakßarí input, using the US International keyboard. Right side gives the output in desired script. Click on the Play button which looks like ▶ . This changes to ❚❚ indicating Active Transliteration ON. Typing anything in the box now, will transliterate to the other pane, the moment change is detected. This interface is designed for whatever font you prefer. Good fonts are suggested, but installing them is your problem.

Choose script from above menu. Left side are Unicode-based schemes. Roman Transliteration and ASCII-based schemes, including non-unicode schemes which require special fonts, are to the right conveniently. Information about chosen script will come in the "About Script" tab. Choosing script is not enough though, you need to choose target Orthography or Language. For rare and obscure scripts, the interface will try to load free fonts if available, for you.

Romanakßarí is my designed scheme for writing Indian languages. Before going on, the letter ß is not a B, as anyone who knows German will attest to. Romanakßarí is a scheme initially designed to take advantage of the Latin-1 encoding scheme of UNICODE, or the Western European ISO-8859-1 scheme. Over time, it has been refined for Sanskrit, Arabic and other languages. It suggests an orthography, which can be used to take advantage of the scheme

Designed initially for Sanskrit, it becomes easy to type using the US-International Keyboard, which means we do not need to install additional software. Also, it can encode features in Sanskrit, which have been scholarly reconstructed, that are not supported by Indian scripts. It also pays more emphasis to systematic grammar, rather than plain phonetics, like Devanágarí does. It allows us to do a LOT more with the language, giving an old language a modern zing!

The purpose of this transliterator is to demonstrate Romanakßarí. It demonstrates the lossless phonetic features of the schemes, and allows me and you, to judge the limits of spelling variations for words in each language. This enables us to encode grammatical cues, while keeping native script information intact and perfectly reproducible.This gives us some amount of spelling freedom. Capital and small letters may be used as well.

Romanakßarí is not a transliteration scheme. What you write is what you mean. Sometimes, many romanakßarí spellings may give the same Devanágarí. Kaliáñam and Kalyáñam give the same कल्याणम् .This is because Devanágarí does not recognize the subtle difference here. Sometimes it does, though. Ácárya and Ácária give आचार्य and आचाय . This difference must have been important in the language (Marathi), because Hindi users don't know about the latter.

Each script supports several potential languages, and each language uses the script differently, with differing shapes for same sounds, different mechanisms etc. The default Generic Mode for each script may not give the desired output expected for the language of your choice. Sanskrit, for instance, has traditionally been written in almost all Indic scripts. As seen in the previous tab, Hindi and Marathi use features of the Devanágarí script in slightly different manner.

Each Language may have many conventions, or options. Sanskrit, for instance: You may turn on or off different types of Sandhi options, output in Pada (separated words) or the more traditional joint form, etc. On choosing a language, the choice will generate a tab with the options for you, so watch the tabs above.

Choosing the right fonts, and installing Support for Complex Scripts is important. This will depend on your operating system. Scripts like Devanágarí, Arabic etc. consist of shapes which change considerably in accordance with their surroundings. These ligatures are compulsary in these scripts, unlike the alphabetic scripts where ligatures are used for Aesthetic reasons. Directions also vary, even within the same script. Arabic writes right-to-left, but numbers are written left-to-right.

One can then install a Pan-Unicode font which support a large number of scripts. Their support for each script varies, and they may not be the best for a particular script, but get their job done. Some fonts like this are Code2000, Gentium, Arial Unicode MS (Microsoft Office). Most operating systems nowadays give good support for most common languages. Specialist fonts pay more attention to details though, and are more accurate, better to look at, and target a specific look and feel, or language.
