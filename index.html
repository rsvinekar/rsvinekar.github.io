A Latin-1 based Indic Transliterator

This was a proof of concept, started in 2008. It is currently outdated, and many iterations which were more feature-full have been lost. But this salvaged copy works, which has been updated in 2014.

The concept was very simple, to write Sanskrit like a modern European language, as in Eastern European languages which bear strong grammatical resemblance to Sanskrit. The idea was to use the Latin-1 scheme.

At that time, fonts supporting systems like IAST were not easy to get, most common fonts did not support the underdot letters. So, why not use the US International Keyboard which was present on Windows and Linux systems?

The retroflex letters could be easily typed by AltGr+letter. So retroflex t is þ, retroflex d is ð, s->ß, even l->ø which visually resembles the devanagari letter. Use acute accent for long letters, the Latin Apex. Again, AltGr+a->á etc.

Vocallic R ( र vs ऋ) and L need not be indicated as context can tell them apart, depending on whether they are flanked by vowels or not. A dummy vowel letter (:) and dummy consonant letter (') has several uses, primarily ऽ and glottal stop.

The concept got extended to various languages. Grammar is highlighted. here ऐ and औ are *not* written as 'ai' and 'au' but as 'ae' and 'ao'. This is because of vrddhi and guna rules. i => guna is a+i = ai = e, vrddhi is a+a+i = a+e = ae.

This makes Sandhi understanding easy. Example ca + eva is caeva , not caiva as in IAST. The letter combination 'ai' is pronounced like ai in the words rain, train, chain etc. The letter x is used for visarga. It may be optionally followed by : as in x: to indicate vowel echo, otherwise it is a silent s, much like french words chateaux, faux, beaux etc. The letter x may be pronounced -sh,-s etc based upon context, just like french -x.

All these points make sanskrit easy to read, we can have capitalization, write sentences in Pada form (not stuck together like traditional writing), but the transliterator can still give you the original Devanagari traditional form, showing that although the input may not look exactly the same as the IAST transcript, it is easier to understand and still preserved all the information needed to put it in traditional form

Submarine

This Project is the Capstone project for the IBM Data Engineering Professional Certificate on Coursera. The data platform architecture of an ecommerce company named SoftCart is designed here. SoftCart uses a hybrid architecture, with some of its databases on premises and some on cloud.

Process:

SoftCart's online presence is primarily through its website, which customers access using a variety of devices like laptops, mobiles and tablets.

  • All the transactional data like inventory and sales are stored in the OLTP database - MySQL server.
  • All the catalog data of the products is stored in the NoSql database - MongoDB server.

SoftCart's webserver is driven entirely by these two databases.

  • Data is periodically extracted from these two databases and put into the Staging Data warehouse running PostgreSQL
  • The production data warehouse is on the cloud instance of IBM DB2 server or Production Data warehouse – PostgresSQL.
  • BI teams connect to the IBM DB2 for operational dashboard creation.
  • Business Intelligence Dashboard - IBM Cognos Analytics is used to create dashboards.
  • SoftCart uses Hadoop cluster as its big data platform where all the data is collected for analytics purposes Data Pipelines - Python ETL.
  • Data Pipelines - Apache Airflow may be used to extract information from log files for monitoring
  • Big data analytics platform – Apache Spark is used to analyse the data on the Hadoop cluster.
